# Jira
#### Étape 1
Prendre son ticket et le bouger à "En cours", avant de travailler dessus. Créer une branche par ticket.
#### Étape 2
Une fois que le ticket est complet, déplacer le ticket à "Review Binôme", et trouver un binôme dès que possible pour le double test. (Possibilité de passer à un autre ticket en attendant de trouver un binôme)
#### Étape 3
Une fois la review binôme terminée, le ticket doit être déplacé dans "Code Review", avant de faire un pull request sur github (après s'être assuré d'avoir push ses derniers changements). Ensuite, le tech lead ratifiera la pull request.
